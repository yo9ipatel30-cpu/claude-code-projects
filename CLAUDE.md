# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository structure

This repo stores outputs generated by Claude Code — currently one project:

- `Outputs/tic-tac-toe.html` — a fully self-contained single-file Tic Tac Toe game (HTML + CSS + JS, no dependencies, open directly in a browser)

## Running the app

No build step. Open the file directly:

```bash
open Outputs/tic-tac-toe.html
```

## Architecture — tic-tac-toe.html

All game logic, styles, and markup live in one file.

- **State**: `board` (9-element array), `current` (active player), `gameOver`, `scores`, `mode` (`'2p'` or `'ai'`)
- **CPU AI** (`bestMove`): greedy heuristic — win if possible, block opponent's win, then prefer center → corners → any open cell
- **Win detection** (`checkWin`): checks `board` state against the 8 `WINS` combos; returns the winning triple or `null`
- **Rendering**: `renderBoard` syncs DOM `.cell` elements to `board` state; no framework, plain DOM manipulation

## Repo conventions

- New outputs go in `Outputs/`. Project-specific subdirectories are created when a project grows beyond a single file.
- `.claude/` is gitignored — local Claude Code settings stay out of version control.

## Git workflow

After completing any meaningful unit of work, commit and push to GitHub. Don't batch unrelated changes into one commit.

Commit message format: short imperative summary in the present tense (e.g. `Add CPU win-blocking logic`, `Fix draw detection bug`). No periods at the end.

```bash
git add <specific files>
git commit -m "Your message here"
git push
```
