## Project: tic-tac-toe.html

Single file. Open directly in a browser:

```bash
open Outputs/tic-tac-toe.html
```

**Architecture**

- **State**: `board` (9-element array), `current` (active player), `gameOver`, `scores`, `mode` (`'2p'` or `'ai'`)
- **CPU AI** (`bestMove`): greedy heuristic ‚Äî win if possible, block opponent's win, then prefer center ‚Üí corners ‚Üí any open cell
- **Win detection** (`checkWin`): checks `board` state against the 8 `WINS` combos; returns the winning triple or `null`
- **Rendering**: `renderBoard` syncs DOM `.cell` elements to `board` state; no framework, plain DOM manipulation

## Repo conventions

- New outputs go in `Outputs/`. Project-specific subdirectories are created when a project grows beyond a single file.
- `.claude/` is gitignored ‚Äî local Claude Code settings stay out of version control.

## Git workflow

Commit and push after every meaningful change so rollback is always possible. Don't batch unrelated changes into one commit.

Commit message format: short imperative summary in the present tense (e.g. `Add headshot to portfolio hero`, `Fix nav active state`). No periods at the end.

```bash
git add <specific files>
git commit -m "Your message here"
git push
```
